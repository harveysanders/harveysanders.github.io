Changing Elements with jQuery

In addition to manipulating the position or existence of elements, jQuery allows us to easily manipulate the attributes and contents of elements. By doing this we can hide, show, add or remove classes, assign arbitrary data, change the styling of these elements, and more.

.attr() and attributes

The core way to modify all elements' attributes is with .attr(), which takes a key and a value as arguments and then updates the element with that attribute.

Let's try it:

1
 
<div class="my-div"></div>

1
2
3
4
5
 
var myDiv = $('.my-div');
myDiv.attr('style', 'background-color: red;');
myDiv.attr('data-whatever', 'arbitrary data');
myDiv.attr('id', 'my-id');
myDiv.attr('class', 'my-new-class');

The result looks like this:

1
 
<div class="my-new-class" id="my-id" style="background-color:red;" data-whatever="arbitrary-data"></div>

As you can see, we can update any attribute on elements that we like. We can also get attributes by not specifying a second argument:

1
2
3
4
 
console.log(myDiv.attr('style')); // background-color: red;
console.log(myDiv.attr('data-whatever')); // arbitrar data
console.log(myDiv.attr('id')); // my-id
console.log(myDiv.attr('class')); // my-new-class

You can also remove attributes entirely with .removeAttr().

Data Attributes

You'll see attributes that look like data-some-name="some-value", like this:

1
 
<div data-id="5"></div>

These are called data attributes and are used to store arbitrary pieces of data in the DOM through the attributes of elements. You could just as easily use .attr() to define an arbitrary attribute to store your data, but using the data-* format will allow you to indicate to yourself and other developers that the attribute is specifically meant to be data. Often times, you'll use data attributes to indicate what element belongs to what conceptually-bound piece of data, such as if you had a set of list items that all correspond to records in a database.

jQuery provides a .data() method that can set arbitrary data on the jQuery objects that wrap specific elements. This is a somewhat complex functionality that won't be covered here. However, if you set a data-attribute on an element manually either in HTML or with .attr(), then you can grab the value of that attribute by using .data() while omitting the data-* prefix:

1
 
<div class="my-div" data-blah="my-blah"></div>

1
2
 
var div = $('.my-div');
console.log(div.data('blah')); // logs "my-blah"

Or:

1
 
<div class="my-div"></div>

1
2
3
 
var div = $('.my-div');
div.attr('data-blah', 'my-blah');
console.log(div.data('blah')); // logs "my-blah"

.attr() vs .prop() (and .val())

There is another jQuery method called .prop() which is very similar to .attr(). .prop() will access the value of properties on the underlying DOM elements. .attr() will access the value of the attributes set in HTML at the time the page was loaded. For instance, take a look at the following example:

1
 
<input type="radio" name="my-radio">

// Check for the "checked" attribute on the input
$('input').attr('checked'); // undefined
// Check for the "checked" property on the input, which all DOM elements
// of type radio or checkbox have
$('input').prop('checked'); // false
// Sets the internal "checked" property of the DOM element to true
$('input').prop('checked', true); // check the radio - we could also click on it
// This returns "checked" because the DOM sets the attribute as checked="checked".
// It's unreliable!
$('input').attr('checked'); // "checked"

Here we can see that .attr() refers to whatever attributes were set on pageload. However, when we interact with an element, its attributes are modified after the fact, and in counterintuitive ways. The real value we want to get is the value of the property of that element.

As a general rule, you should use .attr() when referring to attributes you've already set in HTML and .prop() for accessing the actual properties on DOM elements (like innerHTML, parentChild, etc.). Most frequently you will need to use .prop() for checked, value, defaultChecked, and defaultValue. There is a syntactic sugar method called .value() which access the value property of elements. You should always use this when trying to get the value from input fields.

Showing, Hiding, and Changing Classes on Elements

Often times you'll need to hide or display certain elements, especially on dynamic, single page applications. You might want to apply certain classes to certain elements as well. There are a couple easy-to-use jQuery methods that help you do that.

.show() and .hide() show and hide elements by updating their inline styles to show and hide the elements targeted by these methods. When elements are hidden, they have display: none; set on their style attribute. When elements are shown, they have display set to whatever the value of display was before the elements were hidden. There's also a useful method called .toggle() which will alternate between showing and hiding your element depending on whether or not it's already hidden or showing!

A lot of the time, we also want to change the class on an element in order to change its appearance. We can use .attr() to change classes, but it changes classes absolutely. If you use .attr('class', 'new-class'), all the old classes will be removed in place of new-class. However, what if you want to add or remove only a specific class? We have .addClass() and .removeClass() for that, which will add singular classes or remove them from the element's list of classes. We can also check for the existence of a class using .hasClass(), which takes a class to check for and returns true if the element has that class, and false if it does not.

Showing and Hiding the Fancy Way

jQuery will allow you to animate the CSS of elements with the .animate() method, which will change the value of a given CSS property over a specified time. For example, if you wanted to change the width of an element to 200px over 2 seconds, you would use the following code:

1
2
3
 
$('div').animate({
    'width': '200px'
}, 2000);

You can even open up your DevTools and try it on this page! You'll see everything get all messed up and will have to refresh.

.animate() is fun, but there are a lot of convenience methods for common animations:

  • .fadeIn() - fade in a set of matched elements over time
  • .fadeOut() - fade out as set of matched elements over time
  • .fadeTo() - fade a set of matched elements to a specific opacity over time
  • .slideDown() - animate an element sliding down over time
  • .slideUp() - animate an element sliding up over time
  • .slideToggle() - toggle between an element sliding up or down over time depending on whether it's visible or not

These will let you accomplish simple fading and sliding animations, which are some of the most common animations you see on the web. If you need a more complex animation, you can use .animate().
